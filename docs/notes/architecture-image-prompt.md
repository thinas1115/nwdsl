# ChatGPT 画像生成用プロンプト: nwdsl システム全体構成図

ChatGPT の画像生成にそのまま貼り付けて、システム全体のアーキテクチャ図を描かせるためのプロンプト。

---

横長(16:9)のフラットデザインの技術アーキテクチャ図を描いてください。白背景、角丸の長方形ボックス、はっきり読める日本語ラベル、余白多め、プロの設計資料風。図全体を上から下へ流れる4つの水平レイヤ帯で構成します。各レイヤ帯には薄い背景色を敷き、左端にレイヤ名を縦書きまたは太字で表示してください。

【レイヤ1: 入力(薄い黄色の帯)】
- 左に大きめのボックス「network.yaml(nwdsl DSL)— 唯一の事実ソース」。中に小さな箇条書きで「sites 拠点 / devices 機器+IF / circuits 回線契約 / clouds 事業者網 / links 接続(4種の線種) / paths 通信経路 / views 描き分け定義」と記載
- その右に人物アイコン2つ:「設計者(手書き)」「AIエージェント(OpenSpec仕様駆動で生成)」から network.yaml へ矢印。AI側の矢印には「JSON Schema+リファレンスを参照して記述」と注記
- 右端に小ボックス「nwdsl.schema.json(エディタ/AI補完用)」を置き、network.yaml と点線で接続

【レイヤ2: コア処理(薄い青色の帯)— Python製CLI「nwdsl」】
左から右へ矢印でつながる4つのボックス:
1. 「loader.py — YAML読込・構文検証」
2. 「model.py — pydanticスキーマ(型=検証=Schema生成の単一ソース)」
3. 「validate.py — 意味的整合性検査(参照解決・線種制約・経路隣接性など約20規則)。エラー時はここで停止」
4. 「graph.py — ビュー解決 → RenderGraph(フォーマット非依存の中間グラフ)。WAN→LANのBFS向き付け/LAG自動束ね/経路ハイライトオーバーレイ」
validate.py から下向きの赤い矢印で「NG: 日本語エラー報告(exit 1)」というラベル付きの小さい赤ボックスへ

【レイヤ3: 出力(薄い緑色の帯)— RenderGraphから分岐】
graph.py から3本+1本の矢印で4つのボックスへ分岐:
1. 「render_d2.py → .d2 → D2バイナリ(ELK)→ SVG/PNG」注記「木構造系で最高品質」
2. 「render_svg.py + svg_layout.py → SVG(内蔵エンジン)」注記「不変条件保証: ノード/ラベル重なりゼロ・線の貫通ゼロ。リング=円環、leaf-spine=2段で描画」
3. 「render_mermaid.py → .mmd」注記「GitHub/Obsidianがそのまま描画」
4. 「tables.py → Markdown表(拠点/機器/IF/回線/接続/セグメント一覧。接続先はlinksから逆引き導出)」
このレイヤの右端に縦長ボックス「生成物はすべて同一ソース由来 → 図と表の不整合が構造的に起きない」を配置し、4ボックスを囲む点線でグループ化

【レイヤ4: 利用・配布(薄い紫色の帯)】
- 「playground(nwdsl serve)— ブラウザUI: YAMLハイライト編集/自動検証+描画/エンジン切替(D2/Mermaid/内蔵SVG)/パン&ズーム/Docsビューア(.mdコピー)」を大きめに。レイヤ2〜3全体と双方向矢印で接続し「編集のたびに再検証・再描画」と注記
- 「設計書(基本設計/詳細設計)— 構成図と表を貼り付け」
- 「Git/CI — validate --strict + 生成物の差分チェック」
- 「Docs — チュートリアル/リファレンス/パターン集/FAQ/ADR」

【全体の装飾】
- 線種の凡例を右下に小さく: 黒実線=構内LAN、青太線=キャリア回線、緑破線=論理隣接、紫破線=VPNトンネル、赤太線=障害時経路ハイライト
- タイトルを上部中央に「nwdsl — ネットワーク構成記述DSL 全体アーキテクチャ」、サブタイトル「YAML 1ソースから構成図・表・経路図を生成」
- 矢印はすべてラベル付き、フォントはゴシック体、色は彩度低めの落ち着いた配色

---

## 使い方メモ

- 1回で理想の図にならない場合は「レイヤ2と3の間の矢印を太く」「文字を大きく」など差分指示を重ねる
- 情報を減らしたい場合はレイヤ1と4を省略し、レイヤ2-3(パイプライン)だけを描かせると密度が下がる
