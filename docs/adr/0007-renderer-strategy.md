# ADR-0007: レンダリングエンジン戦略 (D2継続 / 自前レンダラの判断基準)

- Status: Accepted
- Date: 2026-07-04

## Context

「レンダリングを既存ソフト(D2)に任せず、理想的なレンダラを自作して置き換えないのか」という問いに対する判断を記録する。判断材料として3拠点・50台・59接続の検証サンプル (examples/scale-50) を作成し、実描画で品質を評価した。

## 実測評価 (scale-50)

- 全社50台図・本社29台図とも、レイヤ構造(WAN上→LAN下)・拠点コンテナ・ラベル分離を維持したまま描画できた。**ラベル衝突ゼロ**(ADR-0005 の対策後)
- 残る品質課題は「エッジの交差」のみ。これはトポロジ固有の性質(コア冗長のフルメッシュ等)によるもので、交差最小化は NP 困難でありヒューリスティックの出来がすべて。ELK (Eclipse Layout Kernel) のヒューリスティックは長年の実績があり、自作で上回るのは非現実的
- これまでに解決したラベル衝突は、いずれも「D2/ELK がエッジラベルの領域を予約しない」ことが根因だったが、ラベルのノード化 (ADR-0005 補遺) で D2 の枠内で構造的に解決できた

## Decision

**当面 D2 を継続する。** 自前レンダラは作らない。理由:

1. レイアウトの中核(層割当・交差最小化・直交配線)は ELK が最も difficult な部分であり、ここを自作しても品質向上は見込めず保守負担だけが増える
2. これまでの品質課題はすべて「D2 への出力の工夫」(エッジ向き付け・ラベルノード化・幅指定)で解決できており、D2 の表現力の限界にはまだ達していない
3. アーキテクチャ上、`RenderGraph`(フォーマット非依存の中間表現)とシリアライザが分離しているため、**将来 render_svg.py を追加するだけで自前レンダラに置き換えられる**。いま決め打ちする必要がない

## 自前レンダラ (render_svg.py) に着手する条件

以下のような「D2 の出力工夫では実現できない要求」が実際に発生した時点で着手する:

- ポート番号を機器の箱の辺上に固定表示する(パッチパネル風の描画)
- バス型・スター型など NW 図特有の配線スタイル指定
- 冗長ペアの強制対称配置、ラック単位のグルーピング表示
- 図の差分ハイライト(構成変更前後の比較図)

その場合も ELK 相当を作るのではなく、「nwdsl のグラフは実質ツリー+少数の横断リンク」という性質を利用した専用レイアウト(BFS 層 = 行、行内は親の重心順)に限定して実装する。

## Consequences

- D2 のバージョンアップ追従が必要(現在 v0.7.1 を .tools/ に固定)
- 描画品質の改善は「graph.py / render_d2.py での出力の工夫」として継続的に行う
