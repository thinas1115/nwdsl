# ADR-0008: 不変条件保証型の自前SVGレンダラ (どのパターンでも破綻しない)

- Status: Accepted
- Date: 2026-07-04

## Context

ADR-0007 の実測で D2+ELK は leaf-spine / リング / 多拠点概要で破綻すると確認した。
当初案は「破綻した3パターンに特化レイアウトを個別実装」だったが、
**「破綻パターンだけ個別対応するのではなく、どのパターンでも破綻しないことが大事」**
という方針指示を受け、設計を改めた。

## 「破綻しない」の定義

「どんなグラフでも美しい」は保証できない (交差最小化は NP 困難)。しかし「破綻」の
実体は美しさではなく**判読性の喪失**であり、その原因は3つに分解できる。これらは
ヒューリスティックではなく**構築的に保証できる不変条件**である。

| 不変条件 | 保証方法 |
|---|---|
| I1: ノード同士が重ならない | スロット割当 (各ノードに幅ぶんの区間を占有させて敷き詰める) |
| I2: ラベルが何とも重ならない | ラベルを領域予約の対象にする: IF ラベルは接続ポートのスロット、回線ラベルは専用帯 (バンド) の区間スケジューリングで配置 |
| I3: 線がノード箱を貫通しない | エッジの経路を「行間チャネル」(ランク間の水平帯) と「コリドー」(箱と箱の間の垂直隙間) に限定する。多段をまたぐ辺はダミー頂点で分割し、全セグメントがチャネル内に収まるようにする (Sugiyama 方式) |

D2/ELK が破綻する根因は I2 (ラベル領域の非予約) と、レイアウト制御が閉じていて
I3 の経路制約を外から与えられないことにある。自前実装ではこの3つを先に固定し、
交差の少なさ (美しさ) はその制約内での最適化として扱う。

## アーキテクチャ

```
RenderGraph (既存・フォーマット非依存)
   │ svg_layout.py
   ├─ トポロジ分析: BFSランク / サイクル検出 / 拠点メタグラフ
   ├─ 戦略選択 (自動):
   │    ・ring: 拠点メタグラフが単一閉路 → 円環配置
   │    ・汎用: Sugiyama (ランク割当→ダミー頂点→重心法順序付け→スロット座標)
   │      ※ leaf-spine は自然に2段化、多拠点概要は行ラップで格子化 (個別実装ではなく汎用の帰結)
   ├─ 拠点コンテナ: 拠点ごとに内部を独立レイアウトし、行ラップで敷き詰め
   ├─ 大域配線: 雲→機器は空域レーン+コリドー経由 (I3)
   └─ ラベル帯: 回線ラベルを区間スケジューリングで帯に割当 (I2)
   │ render_svg.py
   └─ SVG 出力 (配色・線種は既存 D2 出力と同一の意味論)
```

- 戦略は**ユーザーが選ぶのではなく自動判別**。判別に漏れたグラフも汎用 Sugiyama が
  受け、不変条件 I1〜I3 は全戦略共通の配置・配線プリミティブで保証される
- 既存の D2/Mermaid 出力は残す (`--engine d2|svg`)。SVG エンジンが全ビューで
  同等以上と確認できた時点でデフォルト切替を判断する

## 実装結果 (同日)

- `svg_layout.py` (エンジン) + `render_svg.py` (SVG出力) として実装。`nwdsl render --format svg`、
  playground のエンジン切替 (自動/D2/内蔵SVG) から利用可能。D2 バイナリ不要
- 全サンプル×全ビュー (sample-corp / complex-lan / scale-50 / stress 4種、計37ケース) で
  不変条件 (ノード重なりゼロ・ラベル×ノード重なりゼロ・キャンバス内包) を pytest で機械検証
- 目視確認: D2 でノックアウトだった **リングは正八角形の円環**に、**leaf-spine は
  スパイン行+リーフ行の扇状**に、**12拠点概要はラベル整列した格子状**に描画された。
  経路ビュー (障害✕/迂回矢印/ホップ注記) も動作
- 判明した注意点と対処: 深い段の機器から空域へ上がる線が同拠点の機器を貫通しうる
  → 拠点ボックスの内側余白をコリドーとして迂回する経路に修正済み

## Consequences

- レイアウトの美しさ (交差数・対称性) は ELK に劣るケースがありうるが、
  判読不能にはならない。美しさの改善は制約内の順序付け改良として継続できる
- ラベル・配線・座標を完全制御できるため、D2 で必要だった回避策
  (ラベルノード化・幅ヒント・記述順依存) はこのエンジンには不要
- 描画依存 (D2 バイナリ) なしで SVG まで到達できるようになる (playground にも有利)
